1. Worum geht es in diesem Dokument?

Die FabLab-Website wurde komplett neu aufgebaut – weg von WordPress, hin zu einer schnelleren, moderneren Lösung. Dieses Dokument erklärt in einfachen Worten:

  • Wo die Website und ihre Daten jetzt gespeichert sind
  • Wer worauf Zugriff braucht
  • Wie man als Admin Inhalte hinzufügt (Workshops, Projekte, Dokumente)
  • Was zu tun ist, falls die Person, die das technisch betreut, einmal nicht verfügbar ist

Hinweis: Dieses Dokument richtet sich bewusst an Personen ohne technisches Vorwissen. Fachbegriffe werden kurz erklärt, wo nötig.

2. Wo lebt die Website jetzt?

Früher lief alles über WordPress – ein einziger Anbieter. Die neue Website nutzt vier spezialisierte, kostenlose Dienste. Das ist langfristig stabiler und nicht von einer einzelnen Person abhängig – vorausgesetzt, die Konten gehören FabLab Bremen und nicht einer Privatperson.

GitHub – der „Bauplan“ der Website

Speichert den gesamten Programmcode. Jede Änderung wird hier gespeichert und kann bei Bedarf rückgängig gemacht werden.

Supabase – die „Datenbank“

Speichert alle Inhalte: Workshops, Projekte, Mitgliederinformationen, Dokumente-Verweise. Verwaltet auch die Admin-Anmeldung und Datensicherheit.

Cloudinary – der „Dateispeicher“

Hochgeladene Dokumente (PDF, Word, Excel, PowerPoint) werden hier gespeichert. Workshops, Projekte und Maschinen Bilder liegen separat bei Supabase.

GitHub Pages – das „Schaufenster“

Hier wird die fertige Website veröffentlicht. Die bestehende Domain (fablab-bremen.org) bleibt erhalten.

3. Wer braucht welchen Zugang?

Dienst Wer braucht Zugriff? Wofür?
GitHub Technisch verantwortliche Person(en) Code ändern, Website aktualisieren
Supabase Technisch verantwortliche Person(en) Datenbank, Admin-Konten verwalten
Cloudinary Technisch verantwortliche Person(en) Datei-Speicher verwalten
Admin-Panel Jeder, der Inhalte postet Workshops, Projekte, Dokumente hinzufügen

Wichtig: Alle vier Konten sollten auf eine FabLab-E-Mail-Adresse registriert sein (z.B. info@fablab-bremen.org), nicht auf eine private E-Mail-Adresse. So bleibt der Zugriff beim Verein, egal wer gerade technisch verantwortlich ist.

4. Neue Admins hinzufügen

Um jemand Neuen Zugriff auf das Admin-Panel zu geben:

  1. Eine Person mit Zugriff auf das Supabase-Dashboard öffnet die Nutzerverwaltung
  2. Dort wird die E-Mail-Adresse und das Passwort der neuen Person manuell hinzugefügt
  3. Das System trägt die Person automatisch in die Admin-Liste ein
  4. Die neue Person kann sich danach im Admin-Panel einloggen

5. Inhalte hinzufügen – die Grundlagen

Workshops und Projekte posten

Im Admin-Panel gibt es Formulare zum Hinzufügen von Workshops, Projekten und Maschinen. Titel, Beschreibung und Bilder werden dort eingetragen – die Website aktualisiert sich automatisch.

Dokumente hochladen

Unter „Dokument hochladen“ im Admin-Panel:

  1. Auf „+ Neues Dokument“ klicken
  2. Datei auswählen (PDF, Word, Excel, PowerPoint oder Bild)
  3. Nach dem Hochladen erscheint ein Link
  4. Dieser Link kann in einen Workshop- oder Projekt-Beitrag eingefügt oder direkt geteilt werden

Erlaubte Dateitypen: PDF, Word, Excel, PowerPoint sowie JPEG-, PNG- und WebP-Bilder. Maximale Dateigröße: 25 MB pro Datei.

Ein Dokument löschen

Beim Löschen erscheint eine Warnung, da der Link möglicherweise noch in einem Beitrag verwendet wird. Nach dem Löschen ist der Link dauerhaft nicht mehr erreichbar – vor dem Bestätigen also sicherstellen, dass die Datei wirklich nicht mehr gebraucht wird.

6. Was, wenn die technische Person nicht verfügbar ist?

Das ist eine wichtige Frage, die sich der Vorstand frühzeitig stellen sollte – nicht erst, wenn es dringend wird.

  • Solange alle Konten (Abschnitt 2) auf eine FabLab-E-Mail-Adresse laufen, kann jede Person mit Zugriff auf dieses E-Mail-Postfach die Passwörter zurücksetzen
  • Der Programmcode liegt vollständig auf GitHub und kann von jeder Person mit den nötigen Kenntnissen weitergeführt werden
  • Es empfiehlt sich, mindestens zwei Personen mit technischem Verständnis zu benennen, die im Ernstfall einspringen können

Diese Website wird derzeit ehrenamtlich von einer Einzelperson betreut. Das ist auf Dauer ein Risiko für den Verein – nicht, weil etwas falsch gemacht wurde, sondern weil jedes Projekt, das nur eine Person versteht, verletzlich ist. Es lohnt sich, langfristig eine zweite Person einzuarbeiten.
